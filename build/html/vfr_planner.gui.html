Module main_window¶

Fenêtre principale de l’application VFR Planner

class vfr_planner.gui.main_window.VFRPlannerGUI(root)¶

Bases : object

Fenêtre principale de l’application VFR Planner.

Paramètres:

root (tkinter.Tk) – Fenêtre racine Tkinter utilisée comme conteneur principal de l’interface graphique.

calculate_route()¶

Calcule automatiquement l’itinéraire de vol.

Appelle la méthode de calcul définie dans l’onglet plan_tab.

check_weather()¶

Vérifie les conditions météo le long de l’itinéraire de vol.

Utilise les informations de vol (date, heure, vitesse) pour déterminer le moment exact de passage sur chaque waypoint et fait appel à l’API météo via WeatherService. Affiche les résultats dans une interface dédiée.

En cas d’erreur, affiche un message à l’utilisateur.

copy_plan()¶

Copie le contenu du plan de vol dans le presse-papier.

Si aucun plan n’est défini, affiche un message d’avertissement.

create_menu()¶

Crée la barre de menu principale de l’application.

Cette méthode configure les menus suivants : - Fichier : opérations sur les projets (nouveau, ouvrir, sauvegarder, exporter, quitter) - Édition : actions générales et préférences - Outils : fonctions utilitaires (calculs, météo, carte, diagnostics) - Aide : accès au guide, raccourcis et à propos

create_widgets()¶

Crée et affiche les widgets principaux de l’application.

Cette méthode instancie les onglets de navigation (aéronef, aéroports, itinéraire, plan de vol), ainsi que la barre d’état. En cas d’erreur lors de la création, une interface minimale est affichée.

export_excel()¶

Exporte le plan de vol au format Excel.

Utilise la méthode d’exportation définie dans l’onglet plan_tab.

export_pdf()¶

Exporte le plan de vol au format PDF.

Utilise la méthode d’exportation définie dans l’onglet plan_tab.

get_project_data()¶

Extrait les données actuelles du projet pour les sauvegarder dans un fichier.

Renvoie:

Dictionnaire contenant les données du projet.

Type renvoyé:

dict

mark_saved()¶

Marque le projet comme sauvegardé.

Retire l’astérisque du titre de la fenêtre si présent.

mark_unsaved()¶

Marque le projet comme ayant des modifications non sauvegardées.

Ajoute un astérisque au titre de la fenêtre si ce n’est pas déjà fait.

new_project()¶

Crée un nouveau projet en réinitialisant tous les onglets.

Affiche une boîte de dialogue pour sauvegarder les modifications en cours si nécessaire. Réinitialise les widgets des onglets Aéronef, Aéroports, Itinéraire et Plan de vol.

on_closing()¶

Gère la fermeture de l’application.

Si des modifications non sauvegardées existent, demande à l’utilisateur s’il souhaite sauvegarder avant de quitter. Peut annuler la fermeture.

setup_bindings()¶

Configure les raccourcis clavier et les événements de l’application.

Cette méthode associe des combinaisons de touches à des actions comme créer un nouveau projet, ouvrir, sauvegarder, copier, etc., ainsi que le changement d’onglet pour détecter les modifications.

setup_window()¶

Configure les propriétés de la fenêtre principale.

Définie le titre, la taille minimale, l’icône (si disponible), et la procédure à exécuter lors de la fermeture de la fenêtre.

show_about()¶

Affiche une boîte de dialogue « À propos » avec les informations de version et description du logiciel.

Montre la version, auteurs, fonctionnalités principales et informations légales.

show_interactive_map()¶

Affiche la carte interactive du plan de vol.

Délègue l’affichage à la méthode show_map de l’onglet plan de vol.

show_preferences()¶

Affiche une boîte de dialogue de préférences.

Actuellement, cette fonctionnalité n’est pas encore implémentée.

show_shortcuts()¶

Affiche une boîte de dialogue listant les raccourcis clavier disponibles.

Cette fenêtre liste les raccourcis pour les actions communes dans l’application, incluant fichier, édition, outils et navigation.

show_user_guide()¶

Affiche une fenêtre contenant le guide d’utilisation de VFR Planner.

La fenêtre présente les étapes principales pour configurer l’aéronef, sélectionner les aéroports, planifier l’itinéraire, calculer et exporter le plan de vol, ainsi que les raccourcis clavier utiles.

show_weather_analysis(analysis, weather_service)¶

Afficher l’analyse météo dans une fenêtre avec timing détaillé.

Ouvre une nouvelle fenêtre affichant un rapport complet sur les conditions météo le long de la route de vol, incluant un résumé de vol, détails par waypoint, analyse des tendances météo et recommandations VFR.

Paramètres:

  • analysis – dict contenant les données d’analyse météo et timing

  • weather_service – instance du service météo, utilisée pour vérifier les conditions VFR

system_diagnostic()¶

Affiche un diagnostic du système dans une boîte d’information.

Montre les versions de Python, Tkinter, VFR Planner, la présence des modules requis, le nombre d’aéroports chargés, la configuration API météo, etc.

test_weather_api()¶

Lance un test de l’API météo intégrée.

Appelle la méthode de test API météo depuis l’onglet plan de vol.

update_status_bar()¶

Met à jour le contenu de la barre d’état.

Affiche l’état de la connexion à l’API météo, le nombre d’aéroports visibles, et un message de statut par défaut.

calculate_route()¶

Calculer l’itinéraire complet avec timing météo corrigé.

Cette méthode récupère les waypoints, les données de l’aéronef et du vol, puis appelle la fonction create_itinerary_from_gui avec la clé API météo. Elle affiche ensuite le plan de vol détaillé avec les informations de timing et met à jour la barre de statut.

Affiche une alerte si moins de 2 waypoints sont présents. Gère les exceptions en affichant un message d’erreur.

Renvoie:

None

show_map()¶

Afficher la carte interactive du plan de vol.

Récupère les waypoints de l’itinéraire, crée une carte centrée, ajoute des marqueurs pour chaque waypoint et trace la route entre eux. Sauvegarde la carte en HTML et l’ouvre dans le navigateur par défaut.

test_api()¶

Tester la clé API météo Tomorrow.io.

Cette méthode effectue un test simple en demandant la météo à un point fixe (Montréal) et affiche un message indiquant si la clé API est valide ou non.

Affiche un message d’erreur en cas d’échec.

Lève:

Exception – si la récupération météo retourne une erreur

class vfr_planner.gui.widgets.CustomWaypointDialog(parent, airport_db: AirportDatabase)¶

Bases : object

Dialogue pour ajouter un waypoint personnalisé.

Permet à l’utilisateur d’ajouter un waypoint soit via des coordonnées personnalisées, soit en sélectionnant un aéroport depuis une base de données.

Variables:

  • result (dict or None) – Contient le waypoint créé après validation, ou None si annulé.

  • airport_db (AirportDatabase) – Base de données des aéroports utilisée pour la recherche.

  • dialog (tk.Toplevel) – La fenêtre de dialogue principale.

  • type_var (tk.StringVar) – Variable contrôlant le type de waypoint (“custom” ou “airport”).

  • coord_frame (ttk.LabelFrame) – Conteneur pour les entrées de coordonnées personnalisées.

  • airport_frame (ttk.LabelFrame) – Conteneur pour la recherche d’aéroport.

  • airport_search (AirportSearchWidget) – Widget de recherche d’aéroport intégré.

cancel_clicked()¶

Ferme le dialogue sans enregistrer de résultat.

ok_clicked()¶

Valide les données saisies et ferme le dialogue.

  • Vérifie la validité des coordonnées si le type est “custom”.

  • Vérifie qu’un aéroport est sélectionné si le type est “airport”.

  • En cas de validation réussie, stocke les informations dans self.result.

  • Affiche un message d’erreur en cas de problème et empêche la fermeture.

Lève:

ValueError – Si les coordonnées saisies ne sont pas convertibles en float.

on_type_change()¶

Met à jour l’interface en fonction du type de waypoint sélectionné.

Affiche les champs de coordonnées personnalisées si “custom” est sélectionné, sinon affiche le widget de recherche d’aéroport.
